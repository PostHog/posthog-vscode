<feature>
  <name>generateFlagTypes: FeatureFlag[] to .d.ts string</name>
  <files>src/services/codegenService.ts, src/test/codegenService.test.ts</files>
  <behavior>
    Core behavior:
    - generateFlagTypes([]) returns a string containing "declare namespace PostHogFlags" with the header comment and empty namespace body
    - generateFlagTypes(flags) filters out flags where deleted === true
    - Each non-deleted flag produces a type entry inside the PostHogFlags namespace

    Boolean flags (no payload):
    - Flag with no `filters.payloads` key -> type is `boolean`
    - Flag with `filters.payloads: {}` (empty object) -> type is `boolean`
    - Flag with `filters.payloads: { "true": null }` -> type is `boolean`
    - Flag with `filters.payloads: { "true": undefined }` -> type is `boolean`

    Primitive payloads:
    - Flag with `filters.payloads: { "true": '"hello"' }` -> type is `string`
    - Flag with `filters.payloads: { "true": '42' }` -> type is `number`
    - Flag with `filters.payloads: { "true": 'true' }` -> type is `boolean`
    - Flag with `filters.payloads: { "true": 'null' }` -> type is `null`

    Object payloads:
    - Flag with `filters.payloads: { "true": '{"theme":"dark","version":2}' }` -> type is `{ theme: string; version: number }`
    - Nested objects up to 3 levels deep produce inline types
    - Nested objects at depth > 3 produce `Record<string, unknown>`

    Array payloads:
    - Flag with `filters.payloads: { "true": '[1,2,3]' }` -> type is `number[]`
    - Empty array `'[]'` -> type is `unknown[]`

    Multivariate flags:
    - Flag with `filters.payloads: { "control": '"a"', "test": '{"x":1}' }` -> type is `string | { x: number }` (union of variant types)
    - If all variants produce the same type, still generate a union (or deduplicated)

    Edge cases:
    - Flag key `"my-flag"` (contains hyphen) -> property name is quoted: `"my-flag"`
    - Flag key `"simple_flag"` (valid identifier) -> property name is unquoted: `simple_flag`
    - Unparseable JSON string in payload -> type is `unknown`
    - Payload value is already a parsed object (not a string) -> use directly without JSON.parse

    Header:
    - First line: `// .posthog.d.ts - generated by PostHog for VSCode`
    - Second line: `// Generated: {ISO timestamp}`
    - Third line: `// Flags: {count}`
    - Fourth line: `// Do not edit manually - re-run "PostHog: Generate Flag Types" to refresh.`
    - The timestamp in the header makes exact string matching impractical for the full output; tests should assert on structure (contains namespace, contains correct types) rather than exact full-string equality
  </behavior>
  <implementation>
    Create src/services/codegenService.ts with:

    1. `generateFlagTypes(flags: FeatureFlag[]): string` - main export
       - Filter `!f.deleted`
       - Build header comment with ISO timestamp and flag count
       - Open `declare namespace PostHogFlags {`
       - For each flag: determine type string via payload analysis
       - Close namespace `}`
       - Return full string

    2. Internal helpers (not exported):
       - `inferType(value: unknown, depth?: number): string` - recursive type inference
         - null -> "null"
         - string -> "string"
         - number -> "number"
         - boolean -> "boolean"
         - array -> `${inferType(arr[0], depth)}[]` or `unknown[]` if empty
         - object at depth < 3 -> `{ key: type; ... }`
         - object at depth >= 3 -> `Record<string, unknown>`
         - default -> "unknown"
       - `extractPayloadValue(raw: unknown): unknown` - JSON.parse if string, pass through otherwise
       - `safePropertyName(key: string): string` - quote if not valid JS identifier
       - `inferFlagType(flag: FeatureFlag): string` - orchestrates per-flag type:
         - Get payloads from `flag.filters.payloads`
         - If no payloads or empty -> "boolean"
         - If single key "true" -> extract and infer that payload value
         - If multiple keys (multivariate) -> union of all variant payload types
         - Deduplicate identical types in the union

    Import only `FeatureFlag` from `../models/types`.
    No `import * as vscode` anywhere in this file.
  </implementation>
</feature>
